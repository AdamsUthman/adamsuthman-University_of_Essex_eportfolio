FORUM DISCUSSION

Instruction

Making reference to the article by Fortino et al. (2015), consider the strengths and weaknesses of designing a metamodel to support object-oriented design of the IoT. Design a smart model equivalent to that presented in Figure 6 which would instead support operation of a driverless car.

Answer

Internet of things (IoT) refers to a physical object that has an internet coneection, e.g a tracker, motion sensor. Additionally, object-oriented programming refers to imitating real life objects in computer programs. This makes OOP the perfect approach in developing programs for IoT devices. One of the major strengths of the metamodel approach is that it aids in identifying how smart objects interact with gadgets and services (Fortino et al., 2015). Additionally, it exhorts programmers to concentrate on the design elements and decide upon them before to the start of the implementation and development process. Finally, these metamodels increase the models' reusability by serving as a blueprint for systems that would be created in the future. Unfortunately, this approach has it's weaknesses and onr of them is that in terms of flaws, I believe the metamodel approach shares many of the same issues as the Unified Modeling Language (UML).  For instance, a significant challenge is the absence of "formal semantics." It can be difficult to follow the metamodel approach because it takes a lot of time, especially for projects that must be completed quickly. As a result, convincing project owners and developers to invest time in the design phase may also prove challenging.

SKILLS/KNOWLEDGE GAINED

  • I learnt about the two test frameworks in python: namely Pytest and Nose.
  • I also learnt that global variable modification is not recommended. An independent function that accepts parameters and returns a value might be created instead.
  • I learnt about UNITEST which is the name of the built-in test library provided by Python for unit tests. It offers a class called TestCase. A subclass of TestCase should be built and testing methods should be written when designing a test. The word "test" should be the first word in method names. The number of test methods that can be written in a test case is unlimited.

REFERENCE

Fortino, G., Guerrieri, A., Russo, W. & Savaglio, C. (2015) Towards a Development Methodology for Smart Object-Oriented IoT Systems: A Metamodel Approach. 2015 IEEE International Conference on Systems, Man, and Cybernetics 1297-1302.
